Hi,

it seems that I did write my last mail related to this webreview only to a 
private mailing list. 

http://schillix-on.berlios.de/cr/schily/patch/

Is an attempt to replace the closed source patch(1) from Solaris that is 
expected to be POSIX compliant with an implementation based on Larry Walls 
last version.

It has been modified to install as /usr/bin/patch with 100% POSIX behavior
and /usr/bin/opatch with the old Larry Wall behavior.

It the environment variable POSIXLY_CORRECT is present, it implements POSIX 
behavior but in addition permits several options from the Larry Wall patch
implementation in addition to the POSIX functionality. This is similar to what 
GNU patch does. GNU patch however has no strict POSIX mode and this is why I 
created the new implementation.

Please send your comments, as the code is waiting since 10 days and as I like to 
add it to our mercurial.
